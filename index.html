This project is a front-end for an email client that interacts with an API to send, receive, and manage emails. Built using JavaScript, HTML, and CSS, the single-page application handles all email functionalities within inbox.js.

Key Features:

  • Send Mail: Users can compose and send emails via a POST request to the API, then automatically load their sent mailbox.
  • Mailbox: Users can view their Inbox, Sent, or Archive mailboxes. Each mailbox is populated by making a GET request to retrieve emails, which are displayed with sender, subject, timestamp, and read status.
  • View Email: When clicking an email, users can see the full content. Emails are marked as read via a PUT request.
  • Archive/Unarchive: Users can archive or unarchive received emails through a button, dynamically updating their mailbox.
  • Reply: Users can reply to emails, with the composition form pre-filled with the original email’s details.

This project is a Twitter-like social network website where users can make posts, follow other users, and like posts. Built using Python, JavaScript, HTML, and CSS.

Key Features:

  • New Post: Signed-in users can create new text-based posts via a text area and submit them to the platform.
  • All Posts: Users can browse all posts from all users in reverse chronological order, displaying the username, post content, timestamp, and number of likes.
  • Profile Page: Users can view individual profile pages, showing the user’s followers, who they follow, and all their posts. Users can follow or unfollow others from the profile page.
  • Following Feed: Signed-in users can view posts only from the users they follow, with functionality similar to the "All Posts" feed.
  • Pagination: Posts are displayed 10 per page, with "Next" and "Previous" buttons for easy navigation through older posts.
  • Edit Post: Users can edit their own posts inline using a text area and save changes without reloading the page.
  • Like/Unlike Posts: Users can like or unlike posts, with the like count updated asynchronously without reloading the page.

Family Cooking Book is a user-friendly web app designed for home chefs to organize and store their favorite recipes. Users can create, edit, categorize (by meal type), and mark recipes as favorites, with a visually engaging flip card interface for easy browsing. Recent and favorite recipes are prioritized for quick access.

Built using Django, JavaScript, HTML, CSS and Pillow, the project includes secure user authentication and dynamic frontend behavior, providing a seamless experience for food enthusiasts.

Key Features:

  • Recipe Management: Create, edit, and categorize recipes (e.g., Breakfast, Main Course, Dessert), with the option to upload images. The recipe edit page is pre-populated using a GET request for ease of use.
  • Favorites: Mark recipes as favorites for faster access.
  • Interactive UI: Flip card interface for a dynamic browsing experience.
  • User Authentication: Only authorized users can add or edit recipes.
  • Responsive Design: Optimized for mobile and desktop devices.
